## Environment Variables Required

Make sure these are set in **Vercel Dashboard → Project Settings → Environment Variables**:

### Required Variables

```env
# Supabase Configuration
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

# Stripe (if using)
STRIPE_SECRET_KEY=your_stripe_secret_key
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=your_stripe_publishable_key
STRIPE_WEBHOOK_SECRET=your_webhook_secret

# Razorpay (if using)
RAZORPAY_KEY_ID=your_razorpay_key_id
RAZORPAY_KEY_SECRET=your_razorpay_key_secret

# Application URLs
NEXT_PUBLIC_APP_URL=https://your-domain.vercel.app
NEXT_PUBLIC_API_URL=https://your-domain.vercel.app/api

# Optional
NODE_ENV=production
```

### Setting Environment Variables in Vercel

1. Go to Vercel Dashboard
2. Select your project
3. Go to **Settings** → **Environment Variables**
4. Add each variable for **Production**, **Preview**, and **Development**
5. Redeploy after adding variables

## Troubleshooting

### If Deployment Fails

1. **Check Vercel Build Logs**
   - Go to Vercel Dashboard → Deployments
   - Click on failed deployment
   - Review build logs for errors

2. **Common Issues:**
   - **Missing Environment Variables:** Add them in Vercel dashboard
   - **Build Errors:** Check TypeScript errors with `npm run type-check`
   - **Dependency Issues:** Verify `package.json` is correct
   - **API Route Errors:** Check `vercel.json` configuration

3. **Redeploy:**
   ```bash
   # Make a small change and push
   git commit --allow-empty -m "Trigger redeploy"
   git push origin main
   ```

### If Pages Don't Load

1. **Check Environment Variables:**
   - Verify Supabase credentials are set
   - Check API URLs are correct

2. **Check Build Output:**
   - Verify `.next` directory is generated
   - Check for TypeScript compilation errors

3. **Check Domain Configuration:**
   - Verify domain is properly linked
   - Check DNS settings if using custom domain
